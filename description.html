About MY APIs

Bored API

I started out this project using Bored API to generate a random activity. This API would be helpful for its intended, use – to offer things to do if you’re bored, especially is you allowed the user to select based on type of activity or etc., or for some type of machine learning project where you see how the AI interprets each task. I just used it to pull an activity alone with GET, but I found that a little bland, so I decided to use another API to make it more interesting.

Contextual Web Search API

The second API I used was a Contextual Wed Search API to search for visual elements. This API returns images from the first page of a web search based on a query. It would be cool to use this for some project where you analyze what web searching offers based on query. For example, you could observe the differences in response to two synonyms. It could help users understand what they should be searching to get the results they want. It could also be an easy way for an AI like chat gpt to generate visual aspects along with an explanation. This is sort of how I used it. I took the random activity that I retrieved from Bored API as the query for the url I passed into feth() to get images from Contextual Web Search. It’s not perfect: sometimes the images are forbidden and don’t show up. I could have parsed the query for the relevant keywords and then only used those in the query string, but I didn’t take the time to do that.
